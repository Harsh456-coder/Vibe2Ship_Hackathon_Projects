# 🏙️ Community Hero — Hyperlocal Problem Solver > A civic-tech platform for citizens of **Meerut Ward 12** to report, verify, track, and resolve local issues — powered by AI-assisted categorization, community upvoting, and real-time dashboards. --- ## 📁 Project Structure ``` community-hero/ │ ├── index.html ← Main HTML file (all 4 pages live here) │ ├── css/ │ └── style.css ← All visual styles for the entire app │ ├── js/ │ ├── data.js ← All data (issues, leaderboard, chart data) │ └── app.js ← All interactivity and rendering logic │ └── README.md ← This file ``` ### Why this structure? - **index.html** — Single page app. All 4 pages (Dashboard, Map, Impact, Leaderboard) are in one file; JavaScript shows/hides them. - **css/style.css** — Separated so designers can edit styles without touching HTML or JS. - **js/data.js** — All data is isolated here. To change an issue, edit only this file. - **js/app.js** — All logic is here. It reads from `data.js` and writes to the DOM. --- ## ✨ Features | Feature | How it works | |---|---| | **Dashboard** | Issue cards with live category/status filters. Click any card to load its full detail in the right panel. | | **AI categorization** | Typing in the report modal triggers keyword detection from `AI_HINTS` in `data.js`. Appears as a purple suggestion box. | | **Community upvoting** | Click the 👍 on any card. Toggle on/off. Issues with 25 verifications get auto-escalated. | | **Map View** | Visual ward map drawn in JavaScript. Hover any pin to see issue details. | | **Impact Dashboard** | Horizontal bar chart, spark charts (month-by-month), and a Chart.js weekly bar chart. | | **Leaderboard** | Podium view for top 3, full ranked table. Your row is highlighted in green. | | **Report Wizard** | 3-step modal: title + AI detection → description + photo upload → success + XP reward. | | **Toast notifications** | Bottom-right popups for votes, submissions, and button clicks. | | **XP Gamification** | Earn XP for reporting (+50 XP) and verifying (+5 XP). Displayed in the topbar. | --- ## 🚀 How to Run Locally You **do not need Node.js, Python, or any server** for basic viewing. ### Option A: Just open the file ``` Double-click index.html ``` Works in Chrome, Firefox, Edge, Safari. ### Option B: Use VS Code Live Server (recommended for development) 1. It looks like: `https://github.com/YourUsername/community-hero.git` Then run: ```bash # Connect this local folder to GitHub git remote add origin https://github.com/YourUsername/community-hero.git # Set the branch name to "main" (modern standard) git branch -M main # Push your code to GitHub! git push -u origin main ``` GitHub will ask for your username and password. > ⚠️ **Note:** GitHub no longer accepts account passwords for push. > You need a **Personal Access Token** instead: > 1. GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic) > 2. Generate new token → check **repo** scope → copy the token > 3. Use this token as your password when Git asks --- ### Step 8 — Enable GitHub Pages (free hosting!) Make your site live on the internet for free: 1. Go to your repository on GitHub 2. Click **Settings** tab 3. Scroll to **Pages** (left sidebar) 4. Under Source: select **Deploy from a branch** 5. Branch: **main** | Folder: **/ (root)** 6. Click **Save** 7. Wait 1–2 minutes, then visit: `https://YourUsername.github.io/community-hero` 🎉 Your platform is now live on the internet! --- ## 🔄 How to Update the Site After Making Changes Every time you edit files and want to update GitHub: ```bash git add . git commit -m "What you changed (e.g. added new issue category)" git push ``` GitHub Pages automatically redeploys within 1–2 minutes. --- ## 🛠️ How to Customize ### Add a new issue Open `js/data.js` and add a new object to the `ISSUES` array: ```javascript { id: 6, title: "Your issue title here", cat: "roads", // roads | water | light | waste catLabel: "Roads", status: "open", // open | in-progress | resolved dist: "0.4 km", age: "1 hour ago", media: "2 photos", severity: "High", // Low | Medium | High | Critical votes: 5, comments: 0, ai: 92, verified: 3, needed: 25, timeline: [ { t: "Jun 27", text: "Issue reported", done: true }, { t: "Pending", text: "Awaiting verification", done: false } ] } ``` ### Change the ward name / city In `index.html`, find: ```html
Meerut · Ward 12
``` And change it to your ward. ### Add a new AI keyword hint In `js/data.js`, add to the `AI_HINTS` object: ```javascript "broken wall": "Infrastructure detected · Severity: Medium · Report to PWD." ``` ### Change colors In `css/style.css`, edit the `:root` variables at the top: ```css --green: #1D9E75; /* brand color — change this to your theme */ ``` --- ## 🔮 Future Roadmap (beyond the prototype) - [ ] Backend with Node.js / Firebase — real data persistence - [ ] User authentication (login with phone OTP) - [ ] Real Google Maps integration with actual geo-coordinates - [ ] Image upload to cloud storage (AWS S3 / Cloudinary) - [ ] WhatsApp / SMS notifications for citizens - [ ] Municipal dashboard for officers to manage tickets - [ ] AI image recognition (actual CV model to detect issue type from photo) - [ ] Multi-language support (Hindi, Urdu) - [ ] PWA (Progressive Web App) — installable on mobile --- ## 📦 Tech Stack | Technology | Purpose | |---|---| | HTML5 | Page structure | | CSS3 (with CSS Variables) | Styling and theming | | Vanilla JavaScript (ES6+) | All logic and interactivity | | Chart.js (CDN) | Weekly bar chart on Impact page | | Tabler Icons (CDN) | All icons throughout the UI | | GitHub Pages | Free deployment / hosting | No frameworks.
